# 交接文档 — RAG 课程助手

## 项目背景

这是 Day1 AI Native 训练营的 RAG 课程助手项目，目标是构建一个基于 course-faq.md 的 CLI RAG 问答助手。

## 完整开发流程记录

### 阶段1：需求对齐与 Spec 设计

**对话过程**：
1. 用户要求写 spec.md，定义项目能做什么、不做什么、怎么验收
2. AI 提出 5 个关键问题：检索方式、LLM 调用、输出格式、拒答策略、多轮对话
3. 用户回答：关键词匹配、mock 服务、纯文本+来源、空时拒答、单轮
4. AI 确认 Top3 返回和空时拒答
5. 用户补充：目标≤5、非目标≥3、验收≥3
6. AI 提供 spec 框架，用户指出已有测试文件，需要补充遗漏场景
7. 最终确定 4 条目标、5 条非目标、7 条验收标准、4 条边界条件

**产出文件**：`docs/spec.md`

### 阶段2：超长问题处理规则

**对话过程**：
1. 用户问系统面对超长问题怎么处理
2. AI 分析当前系统无超长问题处理，提出三个方案：截断后检索、直接拒答、不限制
3. 用户选择"截断到指定长度后检索"
4. AI 建议 200 字符，用户改为 300 字符

**产出文件**：更新 `docs/spec.md` 边界条件第 4 条

### 阶段3：数据流设计与接口签名

**对话过程**：
1. 用户要求画出 RAG 六环节数据流，标注每个环节的输入/输出

3. AI 推导 retrieve 和 answer 接口签名
4. 用户确认切片策略（按 ## [faq-XX] 切片）和接口签名

**产出文件**：`docs/design.md`

### 阶段4：Design.md 完善

## 未解决问题详解

### 问题1：测试8-9 混淆问题未拒答

**现象**：
- "说一个Day1没有教的内容" → 应该拒答，但返回了结果
- "RAG除了关键词检索还有什么检索方式？" → 应该拒答，但返回了结果

**原因分析**：
- 测试8的关键词 {'一个', '没有', 'day'} 匹配到了多个 chunk
- 测试9的关键词 {'关键', 'rag', '检索', '关键词', '键词'} 匹配到了多个 chunk
- 当前停用词列表不够完整，'一个'、'没有'、'关键'等词未被过滤

**可能解决方案**：
1. 增加更多停用词（'一个'、'没有'、'关键'、'检索'、'方式'等）
2. 提高 MIN_SCORE 阈值（从 1 提高到 2 或 3）
3. 使用更智能的语义匹配（而非纯关键词匹配）
4. 添加"资料外"关键词检测（如"没有教"、"除了"等）

### 问题2：测试4 跨段落关键词匹配失败

**现象**：
- "Spec中的非目标和AI过度设计有什么关系？" → 预期来源 [faq-04], [faq-10]
- 实际返回 [faq-10], [faq-04], [faq-09]
- 关键词匹配失败（预期包含 '非目标', '过度设计', '数据库', 'Web UI'）

**原因分析**：
- LLM 返回的回答内容不包含预期关键词
- 测试用例的预期关键词可能需要调整

**可能解决方案**：
1. 优化 Prompt 模板，要求 LLM 必须包含特定关键词
2. 或调整测试用例的预期关键词

## 代码逻辑说明

### retrieve.py

```python
# 1. load_chunks: 按 ## [faq-XX] 切片
# 2. extract_keywords: 2-gram/3-gram/4-gram 分词 + 停用词过滤
# 3. retrieve: 关键词匹配，返回 Top 3（MIN_SCORE=1）
```

### answer.py

```python
# 1. 空输入检查：返回"请输入您的问题"
# 2. 超长问题截断：超过 300 字符截断
# 3. 检索：调用 retrieve(question, chunks)
# 4. 拒答：检索结果为空时返回"资料中没有找到依据"
# 5. Prompt 拼接：系统指令 + 上下文 + 用户问题
# 6. LLM 调用：调用 mock API
# 7. 来源引用：回答末尾添加"来源: [faq-XX]"
```
