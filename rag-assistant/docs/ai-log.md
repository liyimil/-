> 每条记录包含五个字段。最关键的是「人工判断」——不是"AI说的对"，而是你采纳/修改/拒绝了AI的什么建议、为什么。

## 第 1 条：Spec 文件设计与创建

| 字段 | 内容 |
|------|------|
| **目的** | 设计并创建 spec.md，明确项目目标、非目标、验收标准 |
| **输入** | 用户要求基于 course-faq.md 构建 RAG 系统，目标≤5、非目标≥3、验收≥3 |
| **建议** | AI 提出 5 个关键问题（检索方式、LLM 调用、输出格式、拒答策略、多轮对话），提供 spec 框架，分析已有测试覆盖的场景 |
| **人工判断** | 采纳核心框架，修改部分细节：关键词匹配（非向量检索）、mock 服务、纯文本+来源、空时拒答、单轮对话；补充验收标准（拒答回复内容、来源引用格式） |
| **验证** | 创建 spec.md，包含 4 条目标、5 条非目标、7 条验收标准、4 条边界条件（含超长问题处理） |
