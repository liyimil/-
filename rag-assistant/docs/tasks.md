## 详细任务说明

### T1：创建项目目录结构

**目标**：创建 rag-assistant 项目的标准目录结构

**验收指标**：
- `src/` 目录存在
- `tests/` 目录存在
- `docs/` 目录存在
- `data/course-faq.md` 存在

**时间估计**：10 分钟

---

### T2：实现 retrieve/answer 函数骨架

**目标**：创建 retrieve.py 和 answer.py 的空函数骨架

**验收指标**：
- `test_basic.py` 的模块加载检查通过
- `test_basic.py` 的接口契约检查通过
- retrieve 函数返回列表
- answer 函数返回 {answer, sources}

**时间估计**：15 分钟

---

### T3：编写集成自动化测试脚本

**目标**：基于 questions.json 编写集成测试脚本

**验收指标**：
- 脚本可运行
- 输出测试结果（通过/失败）
- 覆盖 10 个测试用例

**时间估计**：30 分钟

---

### T4：实现切片功能

**目标**：将 course-faq.md 按 `## [faq-XX]` 切片

**验收指标**：
- 切分出 10 个 chunk
- 每个 chunk 有 id 和 text 字段
- id 格式为 `[faq-XX]`

**时间估计**：20 分钟

---

### T5：实现检索功能

**目标**：实现关键词匹配检索，返回 Top 3

**验收指标**：
- 输入问题，返回 3 个相关 chunks
- 匹配分数降序排列
- 资料外问题返回空列表

**时间估计**：30 分钟

---

### T6：实现 Prompt 拼接和 LLM 调用

**目标**：拼接 Prompt，调用 mock LLM API

**验收指标**：
- Prompt 包含系统指令、上下文、用户问题
- 调用 mock API 返回回答
- 回答内容非空

**时间估计**：20 分钟

---

### T7：实现拒答规则

**目标**：检索结果为空时拒答

**验收指标**：
- 资料外问题返回"资料中没有找到依据"
- 拒答回复包含指定文案
- sources 为空列表

**时间估计**：15 分钟

---

### T8：实现来源引用

**目标**：回答末尾添加来源引用

**验收指标**：
- 回答末尾包含 `来源: [faq-XX]`
- 来源编号与检索结果一致
- 无来源时 sources 为空列表

**时间估计**：15 分钟

---

### T9：实现超长问题处理

**目标**：超长输入截断到 300 字符后检索

**验收指标**：
- 300 字符以内：正常检索
- 300 字符以上：截断后检索
- 截断不影响检索结果

**时间估计**：15 分钟

---

### T10：运行测试脚本，生成测试报告

**目标**：运行集成测试，生成测试报告

**验收指标**：
- 10 个测试用例全部通过
- 生成测试报告文件
- 报告包含测试结果和统计

**时间估计**：20 分钟

---

### T11：运行 test_basic.py + check-submission.sh

**目标**：最终验收

**验收指标**：
- `test_basic.py` 全部通过
- `check-submission.sh` 评级 PASS
- 6 类证据文件齐全

**时间估计**：15 分钟
