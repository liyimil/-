# 全组任务清单

本清单用于在当前分支跑通 A/B/C/D 主链路。所有演示数据均使用 `DEMO-*`，不提交企业样例数据集。

## A：模拟告警数据生成与感知预处理

- 修改目录：`src/alarm_generator/`、`src/perception_agent/`
- 输入：内置 DEMO 告警模板
- 输出：`structured_alarms`
- 验收：能够生成告警序列，并解析出 `alarm_time`、`station_name`、`object_name`、`action`

## B：SKILL 规则管理与智能匹配

- 修改目录：`src/skill_engine/`
- 输入：`structured_alarms`
- 输出：`skill_match`，包含候选规则和 `signal_mapping_states`
- 验收：能够根据告警文本命中特征模板，并输出 feature 内部 signal_mapping 状态
- 当前进展：支持内置 DEMO，也支持从本地 `features.json` / `rules.json` 加载全量 Feature/Rule 定义。

## C：规则分析与表达式引擎

- 修改目录：`src/expression_engine/`
- 输入：`skill_match`
- 输出：`expression_result`，包含 `feature_states`、`feature_hits`、`rule_results`
- 验收：能够求值 `feature.expression` 与 `rule.expression`，触发 rule_id=5 的演示事件
- 当前进展：支持普通布尔表达式、`@n=1/@n=0`、取反、括号，以及 `[2,(@1,@2,...)]` 组合阈值表达式；单条脏表达式会记录失败原因，不中断批量规则处理。

## D：事件生成、调度编排与可视化

- 修改目录：`src/orchestrator/`、`src/event_generator/`、`frontend/`
- 输入：A/B/C 输出
- 输出：标准事件和多页面前端看板
- 验收：能够通过 orchestrator 串联 A/B/C/D，前端提供总览、编排、事件、规则、告警页面

## 测试目标

- Python 语法检查通过
- A/B/C/D 编排命令通过
- 本地样例存在时，`--sample-dir data/samples/samples-md` 可批量处理 `rules.json` 中的全量规则
- 前端 HTML 与 JS 语法检查通过
- 仓库最新文件列表不包含 `data/samples/samples-md/*.json`
