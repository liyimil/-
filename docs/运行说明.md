# 运行说明

本文档用于说明当前项目如何在本地启动后端编排服务和前端页面。

## 1. 安装依赖

在项目根目录执行：

```bash
python -m pip install -r requirements.txt
```

如需使用真实 QwenPaw 编排模式，请确保已安装 `qwenpaw`。当前 `requirements.txt` 已包含项目所需版本。

## 2. 启动前后端联调服务

如果本机已经完成 QwenPaw 模型配置，可使用真实 Agent 编排模式：

```bash
python src/web_server/server.py --adapter real --sample-dir data/samples/samples-md --port 8000
```

如果只是本地演示主链路，或本机尚未配置 QwenPaw 模型，使用离线可运行模式：

```bash
python src/web_server/server.py --adapter demo --sample-dir data/samples/samples-md --port 8000
```

启动后访问：

```text
http://127.0.0.1:8000
```

说明：

- `--adapter real`：使用 QwenPaw Agent 通过工具调用方式运行 A/B/C 主流程；要求本机已安装并配置 QwenPaw active model。
- `--adapter demo`：不依赖模型配置，按本地确定性流程运行 A/B/C 主流程，适合离线演示和调试。
- `--sample-dir data/samples/samples-md`：读取本地样本数据目录中的 `alarms.json`、`features.json`、`rules.json`。
- `--port 8000`：前端和接口服务端口，可按需修改。

## 3. 前端交互

前端启动后会自动请求：

```text
GET /api/dashboard
```

点击页面右上角“刷新分析”会请求：

```text
POST /api/run
```

该接口会重新执行告警感知、SKILL 匹配、规则判定和标准事件生成流程，并刷新页面展示结果。

## 4. 单独运行后端编排

如果只想在命令行查看编排结果，真实 QwenPaw Agent 模式为：

```bash
python src/orchestrator/orchestrator.py --adapter real --sample-dir data/samples/samples-md
```

离线演示模式为：

```bash
python src/orchestrator/orchestrator.py --adapter demo --sample-dir data/samples/samples-md
```

## 5. 数据提交说明

企业样本数据目录：

```text
data/samples/samples-md
```

只在本地读取，不提交到 GitHub。
