# 课题分工与接口规范

整理日期：2026-07-07

## 1. 分工总览

| 课题 | 名称 | 核心职责 | 输出 |
|---|---|---|---|
| A | 模拟告警数据生成与感知预处理 | 生成和清洗告警数据 | 结构化告警序列 |
| B | SKILL 规则管理与智能匹配 | 管理规则 SKILL，匹配候选规则 | 候选 SKILL |
| C | 规则分析与表达式引擎 | 解析规则表达式，判断事件触发 | 规则判定结果 |
| D | 事件生成、调度编排与可视化 | 编排流程，生成事件，前端展示 | 标准事件和看板 |

## 2. 总流程

```text
A 生成结构化告警序列
  -> B 根据特征匹配候选 SKILL
  -> C 加载 SKILL 并解析表达式
  -> C 输出规则触发结果
  -> D 编排流程并生成标准事件
  -> D 前端展示结果
```

## 7. SKILL 文件规范

```markdown
# SKILL: 事件名称

## metadata
- skill_id:
- type:
- domain:
- priority:
- related_alarm_codes:

## description
说明该 SKILL 识别什么事件。

## trigger_conditions
列出触发条件。

## expression
规则表达式。

## output
事件类型、事件等级、推送优先级。

## explanation
业务解释。
```

## 8. 表达式语法约定

第一版支持：

```text
A & B
A | B
!A
A before B
A within 30s
A & B on same_device
count(A) >= 3 within 60s
```

优先实现：

```text
A & B & C within Ns on same_device
```

## 10. 推荐认领策略

如果目标是秋招 Agent 开发岗，优先级建议：

1. C：规则分析与表达式引擎。
2. D：事件生成、调度编排与可视化。
3. B：SKILL 规则管理与智能匹配。
4. A：模拟告警数据生成与感知预处理。

最推荐组合：

```text
主责 C 或 D
参与 B 的 SKILL 规范设计
```

这样最容易在简历中突出 Agent 工作流、SKILL 动态加载、规则推理、结构化输出和企业业务场景。
